# 新项目开发指南

这份指南基于当前工作区 (`/Users/yang/Desktop/Project`) 的最佳实践整理，旨在帮助你快速、规范地启动新的 C# 项目。

## 1. 项目准备 (Preparation)

在开始编写代码之前，请按照以下步骤规划你的项目：

1.  **确定项目目标**：
    *   是工具类（如 `Calculator`）？
    *   是管理系统（如 `HR_System`）？
    *   还是 Web 服务（如 `MyFristApi`）？
2.  **创建独立目录**：
    *   始终在 `Project` 根目录下为新项目创建一个新文件夹。
    *   命名建议使用 PascalCase（大驼峰命名法），如 `SmartLibrary`, `FitnessTracker`。
3.  **初始化项目**：
    *   打开终端，进入新文件夹。
    *   运行初始化命令：
        *   控制台应用：`dotnet new console`
        *   Web API：`dotnet new webapi`

## 2. 代码编写流程 (Coding Workflow)

参考你现有的优秀项目（如 `IBankProject`, `Library_System`），建议遵循以下开发顺序：

### 第一步：定义接口 (Define Interfaces)
*思考：这个系统中的对象有什么共同行为？*
*   **做法**：创建以 `I` 开头的文件。
*   **示例**：如果是交通工具项目，先创建 `IVehicle.cs`，定义 `Start()`, `Stop()` 方法。
*   **参考**：`ICalculator.cs`, `IWorkReport.cs`.

### 第二步：创建领域模型 (Create Domain Models)
*思考：系统里有哪些具体的实体？*
*   **做法**：创建具体的类文件，实现第一步定义的接口。
*   **原则**：**一个类一个文件**。不要把所有类都塞在 `Program.cs` 里。
*   **示例**：创建 `Car.cs`, `Bike.cs` 并继承 `IVehicle`。
*   **参考**：`Developer.cs`, `Manager.cs` (HR_System).

### 第三步：编写核心逻辑 (Business Logic)
*   如果是 Web API：
    *   在 `Models` 文件夹中定义数据结构。
    *   在 `Controllers` 文件夹中编写 API 接口 (参考 `BankController.cs`)。
    *   确保数据库上下文 (`DataContext`) 配置正确。
*   如果是控制台应用：
    *   在类的方法中实现具体的算法或业务流程。

### 第四步：组装与运行 (Main Execution)
*   **做法**：回到 `Program.cs`。
*   **任务**：实例化你的类，调用接口方法，处理用户输入。
*   **调试**：使用 `dotnet run` 运行项目，观察控制台输出或 Swagger 页面。

## 3. 文件命名规范 (Naming Conventions)

保持当前目录的良好风格：

*   **项目文件夹**：`Credit_Card_System`, `Product` (大驼峰)
*   **类文件**：`Employee.cs`, `BankStatement.txt` (大驼峰)
*   **接口文件**：`I` + 名称 + `.cs` (如 `ISwitchable.cs`)
*   **特殊文件**：
    *   文档说明使用 `readme.md` 或 `project_summary.md`。
    *   Web 配置使用 `appsettings.json`。

## 4. 检查清单 (Checklist)

每次提交代码前，请检查：
- [ ] 是否有多余的 `using` 引用？
- [ ] 类名和文件名是否一致？
- [ ] 所有的接口方法是否都已实现？
- [ ] 代码是否可以通过 `dotnet build` 编译成功？
